Kelebihan dan kekurangan bahasa pemprograman C/C++


Kelebihan:

  1. Banyak kompiler dan perpustakaan untuk bekerja dengan [C ++]
  2. Mudah mengakses objek yang diblokir atau disembunyikan oleh bahasa pemrograman lain [C]
  3. Eksekusi program lebih cepat daripada kebanyakan bahasa pemrograman [C / C ++]
  4. Membentuk dasar untuk memahami bahasa pemrograman yang lebih kompleks [C / C ++]
  5. Bahasa pilihan untuk pengembangan aplikasi multi-perangkat, multi-platform [C ++]
  6. Menawarkan portabilitas yang sangat baik [C]
  7. Bahasa berorientasi prosedur dengan sekelompok modul fungsi dan blok.
  8. Ini membuat debugging, pengujian, dan pemeliharaan program lebih mudah [C]
  9. Program lebih efisien dan mudah dipahami [C / C ++]
  10. Pustaka fungsi kaya [C ++]
  11. Berjalan dekat dengan perangkat keras sistem dan karenanya, menawarkan tingkat abstraksi yang rendah [C / C ++]
  12. Mendukung penanganan pengecualian dan kelebihan fungsi [C ++]
  13. Beragam domain aplikasi, seperti game, aplikasi GUI, dan simulasi matematika waktu nyata [C ++]

kekurangan:

  1. Sintaksis kompleks [C / C ++]
  2. Tidak mendukung namespace program [C]
  3. Tidak mampu memecahkan tantangan pemrograman modern dan dunia nyata [C]
  4. Sistem berorientasi objek yang kurang efisien dibandingkan dengan bahasa pemrograman berbasis OOP lainnya [C ++]
  5. Perlu secara manual membuat konstruksi tingkat tinggi [C]
  6. Tidak ada pengumpulan sampah atau alokasi memori dinamis [C / C ++]
  7. Tidak ada pemeriksaan run-time [C / C ++]
  8. Tidak ada pemeriksaan tipe ketat [C]
  9. Bukan pilihan pertama yang mudah untuk belajar pemrograman [C / C ++]
  10. Diganggu oleh masalah buffer overflow dan kerusakan memori [C / C ++]
  11. Perpustakaan standar yang lebih kecil [C]
